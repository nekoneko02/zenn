# はじめに

IPAのネットワークスペシャリスト（NWスペ）・データベーススペシャリスト（DBスペ）を取得済みの新卒3年目（AWS実務経験ほぼなし）が、AWS Solutions Architect Professional（SAP）に約1ヶ月で合格した体験記です。

この体験記の特徴は、以下です：

- IPAで基礎理解があるので、AWSのサービスや概念の理解がスムーズ
- 教科書を1周した時点でUdemy問題集の正解率が68%
- ただし、AWS固有の細かい仕様の暗記は必要

「IPA上位資格を持っているが、SAPに挑戦しようか迷っている」という方の判断材料になれば幸いです。

# 想定読者

- IPA NWスペ・DBスペ等の上位資格を持っている
- AWSの実務経験はほぼないか、触り程度
- AWS SAPの受験を検討しているが、難易度感を測りかねている

# 結論

12/14に教科書を購入し、1/10に受験して合格しました。約1ヶ月[^1]の学習期間です。

[^1]: 年末年始休みを挟んでいるため、1ヶ月とは言え時間は取れた方だと思います。

最も重要な点は以下です：

> **教科書を1周した時点で、Udemy問題集の1回目の正解率が68%だった**

その後、約1ヶ月かけて教科書読み→Udemy問題集の演習を進め、わからないことはChatGPTに壁打ちして整理する、という作業を繰り返しました。本番直前のUdemy 2回目で70%、公式模擬試験でも70%（7/10）を取って、無事に受験・合格しました。

IPA NW/DBスペ持ちにとって、AWS SAPは **「教科書1周＋IPA基礎で合格水準付近に届き、残りはUdemy演習で詰める」** という構造で短期合格が可能でした。

# AWS SAPはどんな試験だったか

本記事を書くにあたって、問題集の内容を振り返りました。

Udemy問題集[^2]のPractice Test計150問を、自分が解いた結果と合わせて独自に分類しました。あくまで筆者1人の整理（※生成AIによる分類）であり、公式の分類ではありません。

## 問題のカテゴリ毎の正誤率

150問を「正解するために主に必要だったスキル/知識のタイプ」で4つに分類し、**各カテゴリの全問数を分母にした誤答率**を計算したものが次の表です。

まとめると、IPAの基礎知識で対応できる問題はあるが、**AWS固有の知識は対策が必要**という構造です。

| カテゴリ（必要なスキル/知識） | 該当問題数 | 不正解 | 誤答率 |
|---|---:|---:|---:|
| A: AWS固有仕様の知識 | 122問 | 43問 | 35% |
| B: 問題文の制約キーワード読み取り | 12問 | 3問 | 25% |
| C: 選択肢の絞り込み（消去法） | 5問 | 0問 | 0% |
| D: アーキテクチャの基礎判断 | 11問 | 0問 | 0% |

それぞれ補足すると、

- **A（AWS固有知識）の誤答率は35%**: 122問中43問のミス。この層の埋め残しが追加学習の中心領域でした
- **B（制約読み取り）の誤答率は25%**: 12問中3問のミスがあり、完璧ではありません。AWS SAPは問題文が長文で、要件の読み落としには注意が必要です。
- **C（消去法）と D（アーキ基礎判断）の誤答率は0%**: それぞれ5問・11問とサンプルは小さめですが、これらのスキルが必要な問題はIPA資格の経験が強く活きると思います。

また、別の観点として、「長時間・長文の試験への対策」です。AWS SAPは試験時間180分・全75問という長丁場の試験で、設問1つ1つも長文です。ただ、IPA試験も長時間・長文ですので、その経験はそのまま活きました。筆者は通しで時間を計った模擬試験はやっておらず、10問だけ時間を計って最低限のペース感覚を確認した程度でしたが、本試験でも特段問題はありませんでした。

## 追加で必要だった学習（3つの層）

「IPA持ちでも追加学習は必要」と書きましたが、その学習内容を整理すると次の3層に分けられます。

### 層1: AWSサービス名と主用途の暗記

最低限必要なのは、AWSの主要サービスの名前と「何のためのサービスか」を知っていることです。

例:

- DataSync = オンプレ↔AWS間のデータ転送サービス
- Transit Gateway = 複数VPCをハブ＆スポークで接続
- Storage Gateway = オンプレからAWSストレージを使うためのゲートウェイ

ここはSAA時点の基礎知識に加えて、教科書でカバーできると思います。

また、ここでもIPAの事前知識が効きます。教科書を読む際に各サービスの「裏側でどんな技術を使っているか」が分かっていると、理解が速くなるためです。余談ですが、Route 53は「DNSのポート番号は53だからRoute 53という名前なのか」と気づきました。

### 層2: サービスごとの細かい仕様・制約

サービス名と主用途を知っているだけでは解けない、細かい仕様の暗記が必要な層です。

例:

- Storage Gatewayのファイルゲートウェイとボリュームゲートウェイの粒度の違い（NFS/SMBで使うかiSCSIで使うか、保存先がS3かEBSスナップショットか）
- AWS DMSがOracle固有機能の一部に非対応
- NLBのクロスゾーン負荷分散を無効化するとAZ間転送料金が削減される

この層は、Udemyの不正解問題の解説を読んだり、ChatGPTに「DataSyncとStorage Gatewayの違いは？」のように質問して整理する作業でひたすら詰めました。

### 層3: NW/DB範囲外のAWS固有サービス

IPAでカバーされない領域、つまりNW/DBの基礎知識が活かせない領域です。具体的には次のあたりです。

- 組織管理: AWS Organizations、SCP、Control Tower、AWS Config
- IAM: IAM Identity Center、許可セット、AssumeRoleとクロスアカウント
- 移行: AWS MGN、AWS DMS、AWS DRS
- エッジ系: CloudFront、Global Accelerator、Lambda@Edge
- メッセージング: SNS、SES、Kinesis（Streams/Firehose）、SQSの使い分け

特に、組織管理やIAMは、IPAの学習だけではカバーされないですし、出題傾向としても多い印象ですので注意が必要です。また、NW系であっても「VPC・ALB/NLB・Route 53」等でChatGPTに複数質問しており、 **AWS固有のNW仕様は別途学習が必要** でした。

## 教科書1周だけでも合格水準に近づける

ここからは1人のサンプルなので断定はできませんが、IPAの資格持ちなら近い状況になる可能性が高いと思います。

教科書1周した後、Udemyの模擬試験の1回目の正解率が68%でした。なお、模擬試験は演習モード（1問ずつ解くモード）で、時間は測らずに取り組みました。

この時点での筆者の状態は次の通りです。

- IPA NW/DBスペの基礎知識あり
- 教科書を読み終えてAWS主要サービスの名前と主用途を一通り把握

この時点で、合格水準に近い結果が出ていました。もちろん、この状態から、特に層2（AWS固有の仕様）の学習をしていく必要はあります。とはいえ、ゼロから学習をするのと比べると、IPAの知識はかなりアドバンテージになると考えられます。

# 学習プロセス

ここからは、筆者の学習プロセスを紹介していきます。最適な学習方法は人それぞれですが、参考になればうれしいです。

## 学習の流れ

```
教科書（黒本）
    ↓
Udemy問題集（複数回）
    ↓
公式模擬試験で力試し
    ↓
本番

※ 各所でChatGPTに壁打ち
```

## 学習記録

| 日付 | 内容 | スコア |
|------|------|------|
| 12/14 | 黒本（教科書）購入・読み始め | - |
| 12/30~1/5頃 | Udemy Practice Test No.1: 1問ずつ解く演習モードで実施 | 68% |
| 1/5~1/10頃 | Udemy Practice Test No.2: 同じく演習モードで実施 | 70% |
| 1/10（午前） | Udemy Practice Test No.3: 最初の10問を時間計測ありで実施<br>公式模擬問題で力試し | どちらも7/10問 |
| 1/10（午後） | 試験本番 | 合格（スコア：813） |

まずは教科書読みからはじめました。平日や休日に少しずつ読み進めました。

次に、年末年始を中心にUdemyの問題集に取り組みました。進め方は、演習モードで1問ずつ解き、わからないところは都度ChatGPTに質問して深掘りをしていきました。ちなみに、演習モードのメリットは、1問ずつ深掘りできること、時間の融通が効きやすいことだと思います。筆者の場合も1日で解き切らず1日10~20問程度ずつで進めていました。

また、通しで時間を計った模擬試験はやっておらず、本番のペース感覚を掴むために10問だけ時間を計って解いた程度でした。年末年始（1/1〜1/5前後）を中心に、前後を合わせて実質約1ヶ月の学習期間です。

# まとめ

IPA NW/DBスペ持ち（実務経験なし）にとって、AWS SAPは1ヶ月程度で合格圏に届く試験でした。

事実として観測できたことは次の3点です。

1. 教科書1周＋IPA基礎の段階でUdemy 68%（合格水準のすぐ近く）
2. 消去法・アーキ基礎判断が必要な問題の誤答率は0%。AWS固有知識が必要な問題の誤答率は35%
3. 学習は「細かい仕様の暗記」と「NW/DB範囲外サービス（IAM/組織/移行系）の学習」に集中

短期合格を狙う場合、教科書を1周して主要サービスの名前と主用途を押さえた後、Udemy等の問題集で細かい仕様を詰める、というシンプルな戦略で届く感触です。

「IPA上位資格を持っているがSAPの難易度感が分からない」という方の参考になれば幸いです。
